Ranking de prioridade por unidade

Defina manualmente o Ranking de cada unidade (ordem/identificação, ex.: ordem de entrega contratual, posição comercial). O valor é salvo automaticamente e passa a valer para novas pendências a partir de agora.

Ranking: campo livre, usado como identificação/prioridade da unidade (ex.: ordem de entrega, posição comercial). Não precisa ser sequencial — você decide o critério.

Alterar o ranking de uma unidade não altera pendências já cadastradas (o valor fica registrado no momento do cadastro) — mas passa a valer para pendências novas a partir de agora.

Dados e backup

Este protótipo roda inteiramente no navegador: todos os dados (cadastros, edições, catálogos e o nome da obra) ficam salvos localmente no localStorage do próprio navegador — nada é enviado a um servidor. Isso significa que os dados persistem se você recarregar a página, mas ficam restritos a este navegador e a este computador. Use o backup para levar os dados para outro navegador/computador ou para guardar um ponto de restauração.
